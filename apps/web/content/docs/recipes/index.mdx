# Recipes

Recipes answer the question: *how do I do X?* Each one is a single sitting — a goal, the canonical code, the gotchas, and a link back to the concept page if you want the deeper read.

Concept pages explain the pieces. Recipes show the pieces wired up.

For route-level agent behavior, start with [Memory](/docs/memory), [Planning](/docs/planning), [Skills](/docs/skills), [Subagents](/docs/subagents), and [Reasoning Effort](/docs/reasoning-effort). Those features are building blocks rather than single recipes today.

## Available recipes

- [Add a tool](/docs/recipes/add-a-tool) — author a tool, get type-safe access from a route
- [Typed state](/docs/recipes/typed-state) — declare state shapes, capture dynamic segments
- [Auth middleware](/docs/recipes/auth-middleware) — short-circuit unauthorized requests
- [Stream output](/docs/recipes/stream-output) — incremental responses via `runs/stream`
- [Retry flaky tools](/docs/recipes/retry-flaky-tools) — recover from transient failures
- [Dispatch from a route](/docs/recipes/dispatch-from-route) — call one route from another
- [Research web UI](/docs/recipes/research-web-ui) — wire a CopilotKit client to the research demo over AG-UI
