# Getting Started

Build a typed AI agent in 60 seconds. Then ship the same code to LangSmith with one command. Dawn is file-system routing for agents — no registry, no tool schemas, no glue.

By the end of this guide you'll have a typed agent at `/hello/[tenant]` that greets a user by name, running locally, ready to deploy.

## 1. Install

```bash
pnpm create dawn-ai-app my-agent
cd my-agent
pnpm install
```

Requires Node.js 22.12 or later.

## 2. Set your API key

The scaffold defaults to `gpt-4o-mini`. Swap providers by editing the `model` string in `index.ts` and exporting the matching key:

<CodeGroup>
```bash title="OpenAI"
export OPENAI_API_KEY=sk-...
```

```bash title="Anthropic"
export ANTHROPIC_API_KEY=sk-ant-...
```

```bash title="Google"
export GOOGLE_API_KEY=...
```
</CodeGroup>

## 3. What you got

Three files. The folder path is the agent endpoint — no registration step.

<CodeGroup>
```ts title="src/app/(public)/hello/[tenant]/state.ts"
import { z } from "zod"

// Route state. The [tenant] segment merges in automatically.
export default z.object({
  question: z.string().default(""),
})
```

```ts title="src/app/(public)/hello/[tenant]/tools/greet.ts"
// Plain TS. Dawn infers input/output types at build time — no Zod, no schema.
export default async (input: { readonly tenant: string }) => {
  return { greeting: `Hello, ${input.tenant}!` }
}
```

```ts title="src/app/(public)/hello/[tenant]/index.ts"
import { agent } from "@dawn-ai/sdk"

export default agent({
  model: "gpt-4o-mini",
  systemPrompt:
    "You are a helpful assistant for the {tenant} organization.",
})
```
</CodeGroup>

Tools under `tools/` are bound to the agent at build time — the LLM picks when to call them. The route's shape is whatever `index.ts` default-exports: `agent`, `workflow`, `graph`, or `chain`.

## 4. Run it

```bash
dawn dev
```

In another shell:

```bash
echo '{"tenant":"acme"}' | dawn run '/hello/[tenant]' --url http://127.0.0.1:<port>
```

The dev server speaks the LangSmith protocol — `/runs/wait` and `/runs/stream`. What works locally works in production.

## 5. Ship it

```bash
dawn build
```

This walks the routes declared via `dawn.config.ts`, runs typegen, and writes `.dawn/build/langgraph.json` plus per-route entry files under `.dawn/build/<routeSlug>.ts`. Hand that directory to LangSmith. Same protocol, same `assistant_id`s, no rewrites.
